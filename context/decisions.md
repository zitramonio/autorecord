# Решения

- 2026-05-06: Создана папка `context/` для хранения проектного контекста.
  - Причина: пользователь запросил структуру для фиксации состояния проекта, задач и решений.

- 2026-05-06: MVP будет Windows-only приложением на .NET WPF с NAudio.
  - Причина: нужен GUI, запись системного вывода и микрофона, фонова работа и автозапуск после перезапуска компьютера.

- 2026-05-06: Встречи читаются из Яндекс.Календаря через приватную iCal-ссылку.
  - Причина: этого достаточно для чтения событий и проще, чем авторизация через аккаунт.

- 2026-05-06: События календаря обновляются раз в час и вручную по кнопке в GUI.
  - Причина: пользователь подтвердил такой режим обновления.

- 2026-05-06: Микрофон и системный вывод сохраняются в один общий `.wav` файл.
  - Причина: пользователю нужен один файл записи встречи.

- 2026-05-06: После настраиваемого периода тишины приложение спрашивает, остановить ли запись.
  - Причина: пользователь хочет вручную подтверждать остановку; при ответе `Нет` повторная проверка выполняется через настраиваемый интервал.

- 2026-05-06: Для реализации подготовлен пошаговый план `docs/superpowers/plans/2026-05-06-autorecord-mvp.md`.
  - Причина: спецификация утверждена, следующий этап — реализация по проверяемым задачам.

- 2026-05-13: Для релизного UX оставлены только модели `gigaam-v3-ru-quality` и `pyannote-community-1`, без выбора модели пользователем.
  - Причина: пользователю нужна стабильная основная связка транскрибации и диаризации, а не тестовый интерфейс сравнения моделей.

- 2026-05-13: Историю транскрибаций в UI не показывать; показывать только текущую транскрибацию и прогресс.
  - Причина: перед релизом важнее простой основной сценарий: записал встречу, получил готовые файлы транскрипта.

- 2026-05-13: Папка по умолчанию для записей и транскриптов — `Documents\Autorecord`.
  - Причина: не засорять корень `Documents` отдельными файлами записей.

- 2026-05-13: В режиме готовности записи открывать только микрофон; устройства вывода открывать только во время записи.
  - Причина: loopback-захват render devices может мешать Bluetooth/USB-аудио устройствам, например Creative BT-L3.

- 2026-05-13: Для записи системного звука использовать только default render endpoints Windows и native/mix format устройства.
  - Причина: не трогать лишние активные output endpoints и не навязывать устройствам формат `48 kHz stereo`.

- 2026-05-13: Корневой ярлык `C:\Projects\autorecord\Autorecord.lnk` должен запускать publish build из `artifacts\publish\Autorecord`.
  - Причина: пользователю нужен один стабильный запускной путь без поиска exe по build-папкам.

- 2026-05-13: Использовать собственную packaged icon для exe, окна и tray icon.
  - Причина: приложение должно нормально выглядеть в трее и не использовать стандартную Windows application icon.

- 2026-05-13: Отменённые транскрибации не показывать как текущую транскрибацию.
  - Причина: `Cancelled` задача с остановленным worker выглядит как зависшая, если остаётся в карточке текущей обработки.

- 2026-05-13: Аварийные `.recording.wav` с нулевыми размерами WAV chunks восстанавливать автоматически.
  - Причина: при принудительном завершении приложения аудиоданные могут сохраниться, но header остаётся не финализированным; пользователь не должен терять запись и возможность транскрибации.

- 2026-05-13: Для текущей транскрибации показывать этапы обработки вместо попытки делать более точный progress bar.
  - Причина: GigaAM/Pyannote workers могут долго работать без детального прогресса; этапы честнее объясняют, где приложение находится, и ниже риск сломать pipeline.

- 2026-05-13: Убрать progress bar и процентную строку из карточки текущей транскрибации.
  - Причина: после добавления этапов линия прогресса занимает место и вводит в заблуждение из-за долгих worker-стадий без granular progress.

- 2026-05-14: Для релиза сделать самораспаковывающийся installer `.exe` с пришитым ZIP payload вместо IExpress/Inno/NSIS.
  - Причина: Inno/NSIS не установлены, IExpress зависает/падает на крупном payload; собственный stub повторяемо упаковывает приложение и bundled модели в один exe.

- 2026-05-14: Вернуть выбор ASR-модели пользователю и добавить `sherpa-onnx-nemo-parakeet-tdt-0.6b-v3-int8` как экспериментальную ASR-модель.
  - Причина: пользователь хотел сравнить Parakeet v3 INT8 ONNX/sherpa с GigaAM; решение позже отменено для публичной версии.

- 2026-05-14: Диаризацию оставить фиксированной на `pyannote-community-1`, а выбор открыть только для ASR.
  - Причина: запрос касался смены модели транскрибации; расширение выбора диаризации усложнило бы релизный UX без текущей необходимости.

- 2026-05-14: Сохранять технические дорожки `*.mic.wav` и `*.system.wav` рядом с итоговым combined MP3.
  - Причина: Quill-подобный подход отделяет локальный микрофон от системного микса без рискованной speech separation; это дает материал для будущего улучшения диаризации, но не ломает текущий pipeline, который продолжает работать с combined MP3.

- 2026-05-14: Для публичной версии убрать Parakeet из каталога и installer, оставить bundled только `gigaam-v3-ru-quality`.
  - Причина: Parakeet слишком тяжелый для публичного дистрибутива; пользователь попросил не предлагать его скачивание.

- 2026-05-14: `pyannote-community-1` не включать в installer; модель диаризации скачивается пользователем с Hugging Face через token.
  - Причина: модель gated на Hugging Face и требует принятия условий доступа пользователем.

- 2026-05-14: Добавить вкладку `О программе` и installer EULA с лицензиями моделей и предупреждением о записи разговоров.
  - Причина: публичный релиз должен явно показывать условия использования моделей и обязанность пользователя уведомлять участников записи и получать необходимые согласия.

- 2026-05-14: Переделать installer в обычный GUI wizard без console window.
  - Причина: консольное окно выглядит как технический артефакт; публичная установка должна идти через стандартные экраны лицензии, выбора папки, прогресса и завершения.

- 2026-05-14: Упростить вкладку `Транскрибация` для публичного релиза.
  - Причина: пользователь не должен видеть внутренний выбор ASR/диаризации, статусы моделей и служебные кнопки; оставить понятную модель `GigaAM v3`, скачивание Pyannote при необходимости и рабочие настройки транскриптов.

- 2026-05-14: По умолчанию включать только TXT-экспорт транскрипта.
  - Причина: остальные форматы пользователь выбирает сам, чтобы не получать лишние файлы.

- 2026-05-15: На главном экране группировать настройки в framed-блоки.
  - Причина: экран записи должен быть легче сканировать; календарь, статус записи, автоостановка и общие настройки имеют разные сценарии использования.

- 2026-05-15: Автостарт записи по календарю сделать явной отключаемой настройкой.
  - Причина: пользователь должен иметь возможность оставить iCal-настройки и ручную запись, но запретить автоматический старт по событиям.

- 2026-05-15: Автоостановку по тишине сделать явной отключаемой настройкой, а timeout бездействия — настраиваемым.
  - Причина: для разных встреч нужен разный уровень автоматизации; фиксированные 2 минуты без ответа недостаточно гибкие.

- 2026-05-15: Всплывающие уведомления сделать отключаемыми.
  - Причина: часть пользователей не хочет получать toast-уведомления в правом нижнем углу во время работы.

- 2026-05-15: Для WinForms installer wizard задавать реальный начальный размер страницы до добавления anchored-контролов.
  - Причина: иначе `AnchorStyles.Right/Bottom` рассчитывает layout от дефолтного размера panel и может уводить RichTextBox за границы окна, перекрывая checkbox согласия.

- 2026-05-15: Если пользователь выбирает `C:\Program Files`, устанавливать в `C:\Program Files\Autorecord`, а для защищённых путей перезапускать installer через UAC.
  - Причина: `Program Files` — контейнер для приложений, а не папка конкретного приложения; без повышения прав Windows блокирует запись файлов.

- 2026-05-15: Public installer должен оставаться одним self-contained `.exe`, несмотря на более медленный старт из-за размера.
  - Причина: сценарий `exe + payload.zip` требует от пользователя скачать и держать рядом два файла, что неприемлемо для публичной установки.

- 2026-05-15: Не добавлять pre-start окно перед запуском большого installer.
  - Причина: задержка self-contained `.exe` происходит до выполнения кода installer, поэтому окно `Установка приложения Autorecord сейчас начнется` не сможет появиться во время этой паузы.

- 2026-05-15: При ошибке авторизации Hugging Face показывать короткий текст `Ошибка - неверный токен`.
  - Причина: пользователю нужен понятный статус сразу на вкладке после попытки скачать `pyannote-community-1`.

- 2026-05-15: Главное окно открывать размером `760x720`, оставить resizable и добавить scroll на вкладку `Запись`.
  - Причина: все настройки должны помещаться по умолчанию, но при меньшем размере окна не должны обрезаться.

- 2026-05-15: Описывать источник событий как любой календарь с iCal/.ics-ссылкой, а не только Яндекс.Календарь.
  - Причина: реализация скачивает произвольный `CalendarUrl` и парсит iCalendar через `Ical.Net`; провайдер календаря не зашит в код.
