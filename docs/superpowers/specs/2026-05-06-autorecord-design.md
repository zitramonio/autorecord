# Autorecord MVP Design

Дата: 2026-05-06

## Цель

Сделать Windows-only приложение, которое читает встречи из Яндекс.Календаря по приватной iCal-ссылке и автоматически записывает звук микрофона и системного вывода во время встреч. Приложение должно работать после перезапуска компьютера без ручного запуска.

## Подтвержденные решения

- Платформа: Windows-only.
- GUI: нативное WPF-приложение на .NET.
- Аудио: NAudio.
- Календарь: приватная iCal-ссылка Яндекс.Календаря.
- Автозапуск: Windows Task Scheduler при входе пользователя в систему.
- Формат записи MVP: один общий `.wav` файл.
- Микрофон и системный вывод сводятся в один файл.
- Имя файла строится по времени начала записи: `дд.мм.гггг чч.мм.wav`, например `06.05.2026 18.42.wav`.

## Поведение календаря

- Приложение скачивает и парсит `.ics` по iCal-ссылке.
- События обновляются автоматически раз в час.
- В GUI есть кнопка ручного обновления календаря.
- Для MVP учитываются только события с конкретным временем начала.
- События на весь день не запускают запись.
- Если событие уже началось до запуска приложения, MVP не стартует запись задним числом.
- Если во время активной записи начинается другое событие, отдельная вторая запись не создается.

## Поведение записи

- В момент начала подходящего календарного события приложение запускает запись.
- Запись захватывает:
  - устройство ввода, выбранное системой по умолчанию;
  - устройство вывода, выбранное системой по умолчанию, через WASAPI loopback.
- Оба потока сводятся в один аудиофайл.
- Приложение отслеживает уровень звука на вводе и выводе.
- Если на обоих источниках нет звука в течение настраиваемого интервала тишины, по умолчанию 1 минута, приложение показывает запрос: выключить ли запись.
- В запросе есть кнопки `Да` и `Нет`.
- Если выбрано `Да`, запись останавливается и файл сохраняется.
- Если выбрано `Нет`, запись продолжается. Приложение ждет настраиваемый интервал повторной проверки, по умолчанию 5 минут.
- Если пользователь не ответил на запрос, запись продолжается; это не считается согласием на остановку.
- После повторного ожидания приложение снова требует непрерывную тишину в течение интервала тишины. Если условие выполнено, запрос показывается еще раз.
- Пока пользователь не подтвердил остановку, запись продолжается.
- Если файл с рассчитанным именем уже существует, приложение не перезаписывает его и добавляет числовой суффикс, например `06.05.2026 18.42 (2).wav`.

## GUI

Основное окно настроек содержит:

- поле для iCal-ссылки;
- кнопку ручного обновления календаря;
- выбор папки сохранения записей;
- режим записи:
  - все события;
  - только события с меткой;
- поле метки для режима записи по метке;
- настройку интервала тишины перед запросом остановки;
- настройку интервала повторного ожидания после ответа `Нет`;
- переключатель автозапуска;
- статус приложения: ожидание, запись, ошибка календаря, последняя синхронизация.

Приложение должно уметь работать из трея. После автозапуска оно стартует свернутым в трей.

## Уведомления и запросы

- При старте записи показывается уведомление `Запись началась`.
- После сохранения файла показывается уведомление `Запись сохранена` с путем к файлу.
- После периода тишины показывается интерактивный запрос остановки записи.
- Если интерактивные кнопки в системном уведомлении недоступны или не работают в окружении пользователя, допустим fallback на небольшое foreground-окно поверх остальных окон.

## Хранение настроек

Настройки хранятся локально на компьютере пользователя:

- iCal-ссылка;
- папка сохранения;
- режим записи;
- метка для фильтрации событий;
- интервал автоматического обновления календаря, фиксированно 1 час для MVP;
- интервал тишины перед запросом остановки;
- интервал повторного ожидания после ответа `Нет`;
- включен ли автозапуск.

Интервалы в настройках должны быть положительными значениями. MVP может хранить их в минутах.

## Компоненты

- `Settings UI`: WPF-окно настроек и статуса.
- `Tray Host`: иконка в трее, запуск окна настроек, фоновая работа.
- `Calendar Sync`: загрузка iCal, парсинг событий, ручное и почасовое обновление.
- `Schedule Monitor`: определение момента старта встречи.
- `Recording Service`: управление жизненным циклом записи.
- `Audio Capture`: захват микрофона и системного вывода.
- `Audio Mixer/Writer`: сведение потоков и запись `.wav`.
- `Silence Detector`: определение тишины на обоих источниках.
- `Stop Confirmation`: запрос остановки после тишины и повторная проверка после ответа `Нет`.
- `Startup Manager`: регистрация и снятие задачи автозапуска в Windows Task Scheduler.

## Ошибки и ограничения MVP

- Если iCal-ссылка недоступна, приложение показывает ошибку в GUI и продолжает использовать последний успешно загруженный список событий, если он есть.
- Если папка сохранения недоступна, запись не стартует, а пользователь получает ошибку.
- Если микрофон или устройство вывода недоступны, запись не стартует, а пользователь получает ошибку.
- MVP не делает распознавание речи, транскрибацию, облачную синхронизацию или загрузку файлов куда-либо.
- MVP не редактирует события календаря.
- MVP не реализует `.mp3`; при необходимости это отдельное расширение после стабильной `.wav` записи.

## Проверка

- Юнит-тесты для парсинга календаря и фильтрации событий.
- Юнит-тесты для логики расписания и выбора ближайшего события.
- Юнит-тесты для логики остановки записи после тишины, ответа `Да` и ответа `Нет`.
- Ручная проверка записи микрофона и системного вывода на Windows.
- Ручная проверка автозапуска после перезагрузки или повторного входа пользователя.
- Ручная проверка уведомлений о старте, запросе остановки и сохранении файла.
